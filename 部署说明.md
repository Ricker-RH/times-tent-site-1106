# 时代篷房网站部署说明

## 仓库与线上地址

- GitHub 仓库：`https://github.com/Ricker-RH/times-tent-site-1106.git`
- 生产网站：`https://www.timestent.com/`
- 后台入口：`https://www.timestent.com/admin`

## 技术栈

- Next.js `14.2.4`
- React `18.3.1`
- PostgreSQL 数据库
- 后台上传图片会在服务端压缩为更轻的 WebP，并通过 `/api/uploads/[id]` 输出

## 本地开发

```bash
npm install
npm run dev -- -p 3001
```

本地访问：

- 前台：`http://localhost:3001`
- 后台：`http://localhost:3001/admin`

## 必需环境变量

这些变量需要配置在 `.env.local` 和线上部署平台的 Environment Variables 中。不要把明文密钥提交到 GitHub。

```bash
DATABASE_URL=postgresql://...
ADMIN_JWT_SECRET=...
TRANSLATION_SERVICE=...
LIBRE_TRANSLATE_URL=...
```

当前本地数据库连接为 PostgreSQL/Neon：

- 用户：`neondb_owner`
- Host：`ep-soft-morning-ah0wbu6k-pooler.c-3.us-east-1.aws.neon.tech`
- Database：`neondb`
- 代码使用 SSL 连接

完整密码在本地 `.env.local` 中，不应写入文档或聊天记录。

## Vercel 部署建议

1. 在 Vercel 中导入 GitHub 仓库 `Ricker-RH/times-tent-site-1106`。
2. Framework Preset 选择 `Next.js`。
3. Install Command 使用默认或 `npm ci`。
4. Build Command：`npm run build`。
5. Output Directory 保持默认，不需要填写。
6. Node.js 版本建议使用 `20.x`。
7. 在 Vercel 的 Environment Variables 中添加上面的环境变量。
8. 绑定正式域名 `www.timestent.com`。
9. 推送到主分支后，由 Vercel 自动构建发布。

## 图片优化与上线注意

历史后台图片已经压缩到约 `149.58MB`，本地可用下面命令重新执行：

```bash
npm run uploads:optimize -- --all --apply --batch-size 8
```

本地或自有服务器可预热图片磁盘缓存：

```bash
npm run uploads:cache -- --all --batch-size 50
```

`.local/upload-cache` 是本地加速缓存，已加入 `.gitignore`，不需要提交。Vercel 等无状态平台可能不会持久保存本地缓存，但代码会自动回退到数据库读取，不影响图片显示。

如果后续希望线上图片继续提速，建议把后台上传图片从数据库 BLOB 逐步迁移到对象存储和 CDN，例如阿里云 OSS、Cloudflare R2、AWS S3 或 Vercel Blob。
